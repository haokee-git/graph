## 项目简介

一个采用 TypeScript 编写的 Web 应用，使图论可视化。

在保证能在服务器上运行的情况下，在其基础上额外编写少量 Electron 代码用来生成 Electron 式桌面程序。

先开发服务器代码，我给出完成命令后再开发 Electron 代码。

## 具体实现

- 采用 Material Design 4.0 的设计语言
- 采用 TypeScript 编写
- 整个页面为白色主题，简洁而美观
- 所有依赖采用 NPM 而非 CDN 引入
- 编写脚本用来启动服务器（开启服务器后终端进程退出，在后台运行）和构建 Electron 式桌面程序。均需要支持 Windows 和 Linux 端。

## 页面布局

分为左右双栏，左侧为输入区，右侧为显示区，底部信息区。

## 左侧输入区

- CheckBox 选择单向边或双向边，默认单向边。
- CheckBox 选择是否硬性规定点数。如果硬性规定点数 n，那么右侧一定显示编号为 1~n 的点，否则只显示编辑器当中有被连边的点。
- 多行 Monaco 编辑器输入边。

## 右侧显示区

- 采用自研物理引擎，模拟点之间的斥力和点和边的弹力，可以通过鼠标拖动某个点的位置，实时更新物理引擎。可能需要外部库来画图。

## 输入格式

每一行表示一条边。一行当中，如有 `a b c`（被空白字符（空格和 Tab）分割）则表示编号为 a 和 b（可能为不含空白字符的字符串）的点之间有边（根据 CheckBox 判断是单向边或双向边，单向边需要为指向的方向增加箭头），边的信息为 c（不含空白字符的字符串）。

注意在 c 不出现（仅输入 `a b`）的情况下，该边不需要显示信息，但是还是需要画线。

一个点是一个圆，圆中央为点的编号（可能为不含空白字符的字符串）。圆的半径最小值为一个合适值，如果编号过长自动根据长度使用较大的半径。

边是一条弹力绳，绳子的两端连接两个点。弹力绳可能会弯掉。你需要真实地模拟这一点。
